# Product Backlog

This is the authoritative list of every user-visible capability CHITTI will possess. We measure progress against this backlog in defined product versions.

## Capability Acceptance Rule
Before implementing any new feature, we must explicitly answer four questions:
1. What user problem does it solve?
2. Which existing folders own it?
3. Which benchmark proves it works?
4. Which backlog item does it complete?
