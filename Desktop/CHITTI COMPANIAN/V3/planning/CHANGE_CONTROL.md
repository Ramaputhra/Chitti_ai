# Change Control Policy

Every future request or new idea must follow this strict triage flow to prevent the project from derailing.

```text
New Idea
   │
   ▼
Is it a bug?
 ├── YES ──▶ Schedule in Current Sprint
 │
 └── NO
      │
      ▼
   Is it a new Feature?
    ├── YES ──▶ Schedule in Next Sprint
    │
    └── NO
         │
         ▼
      Is it an Architecture Change?
       ├── YES ──▶ Requires Approval (Architect Guardian / Project Director)
       │
       └── NO
            │
            ▼
         Is it a Future Vision?
          └── YES ──▶ Add to Backlog
```

This document is the highest authority on change management. Never interrupt an active sprint for a Feature or Future Vision item.
