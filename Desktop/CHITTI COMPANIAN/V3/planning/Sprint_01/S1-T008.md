# Task ID: S1-T008

**Title:** Event Bus
**Owner:** Desktop Lead
**Priority:** Critical
**Dependencies:** S1-T004, S1-T007

## Description
Implement the `IEventBus` and `EventBus`. This is the core in-process communication backbone of the system.
Supports `Event` objects, four EventTypes (Event, Command, Request, Response), Priorities, and thread-safe routing.
Automatically logs all published events to the LoggingService.

## Acceptance Criteria
- `IEventBus` and `Event` classes defined.
- `EVENT_CATALOG.md` and `EVENT_NAMING_STANDARD.md` updated with strict namespaces.
- Pub/Sub supports `subscribe`, `subscribe_once`, `unsubscribe`.
- Thread safety verified.
- Exception isolation (one failing subscriber does not crash the bus).
- Performance requirement (<5ms for 100 subscribers) verified in tests.
