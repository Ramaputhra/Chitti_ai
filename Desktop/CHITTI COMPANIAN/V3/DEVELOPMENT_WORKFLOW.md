# Development Workflow

This document defines exactly how every sprint works. There are no exceptions to this flow.

**Product Owner** (User)
↓ Defines the next experience
**Sprint**
↓ Planning and scoping the required capabilities
**Implementation**
↓ Building the feature within the frozen architecture
**Tests**
↓ Verifying the implementation
**Architecture Review**
↓ Ensuring compliance with Project Constitution and Frozen Architecture
**Merge**
↓ Code is integrated and Live Demonstration is presented
**Next Sprint**
